Task 1: Greedy Snake

This is a small programming exercise for you to get familiar with Python, which is a dynamically typed language. In this task, you have to strictly follow our proposed OO design and the game rules for "Greedy Snake" stated in this section.

Description

Greedy Snake is a single-player game. When the game starts, the player uses the 4 keys ('w', 'a', 's', and 'd') to move a snake around the board, represented by a 22 x 22 map. After inputing the keys, you should press the “Enter" key to proceed. If the key 'w' is pressed, the snake will move up in the next step. If the key 's' is pressed, the snake will move down in the next step. If the key 'a' is pressed, the snake will move left in the next step. If the key 'd' is pressed, the snake will move right in the next step. If you press any of the keys other than these four, the snake will move right by default.

As the snake finds food, it eats the food, and thereby grows longer. We have two different kinds of food here (i.e., apple and water) and each food has a base score (equals to 100), while the actual score the snake can obtain from the food is determined by the food attributes. Specifically, the score of an apple is determined by its maturity. If it is mature, then the actual score is equal to the base score. Otherwise, the actual score would be 0. The score of water is determined by its volume, which is equal to the volume * base score. The final score obtained by the player is calculated based on the number and the type of food your snake ate. The game ends when the snake either crashes into a wall or moves into itself. The goal is to make the snake grow up to the maximum length (set as 15 by default). This means you win the game.

Preparation

Input Specification

In this exercise, you are required to use command line to get input information. There is only one thing requiring user-input, which is the moving direction of the snake. Input one of the pre-defined 4 keys ('w', 'a', 's', and 'd') and press "Enter" to control the direction.

Move and Display Game Information

The snake always starts from the top-left corner of the board. You can use 4 keys ('w', 'a', 's', and 'd') on the keyboard to control the movement of the snake ('w' means up, 's' means down, 'a' means left, 'd' means right. Right is the default value.). If an invalid key (a key not in the 4 pre-defined keys) is pressed, the snake will move to the right by default.

After each movement, the current score obtained by the player will be printed on the board.

The snake's head and body are represented by '@' and '*', respectively. 'o' represents the apple and '!' means water. The boundary is represented by '#'.

Game End

You lose the game and the game is over when the snake either crashes into a wall or moves into itself.

On the other hand, you win if your snake grows up to the maximum length, which is set as 15 by default.

Duck Typing

The following synopsis of Duck Typing is summarized from:

  • http://en.wikipedia.org/wiki/Ducktyping
  • http://www.sitepoint.com/making-ruby-quack-why-we-love-duck-typing
In standard statically-typed object-oriented languages, objects’ classes (which determine an object’s characteristics and behavior) are essentially interpreted as the objects’ types. Duck Typing is a new typing paradigm in dynamically-typed (late binding) languages that allow us to dissociate typing from objects’ characteristics. It can be summarized by the following motto by the late poet James Whitcombe Riley:
When I see a bird that walks like a duck and swims like a duck and quacks like a duck, I call that bird a duck.
 The basic premise of Duck Typing is simple. If an entity looks, walks, and quacks like a duck, for all intents and purposes it is fair to assume that one is dealing with a member of the species anas platyrhynchos. In practical Python terms, this means that it is possible to try calling any method on any object, regardless of its class. An important advantage of Duck Typing is that we can enjoy polymorphism without inheritance. An immediate consequence is that we can write more generic codes which are cleaner and more precise.
